工具栏布局方案

当前 5 个文字按钮已显拥挤，新增 New Window / RUA / Config 后会更严重

方案 A：图标工具栏（推荐）

用矢量图标代替文字，紧凑优雅

📄 📂 💾 │ 🔲 │ ⌨️ Win ⓘ

优点

  • 最节省空间，一行放所有按钮
  • 视觉清晰，图标直观
  • 缩放性好，加新功能不影响布局

代价

  • 需引入图标或使用 Unicode 符号
  • 图标需要用户学习成本

方案 B：经典菜单栏

File / View / Help 多级菜单，无限扩展

File View Help
▸ File: New · Open · Save · Save As · New Window
▸ View: Key Mode (Win/Mac)
▸ Help: About (RUA) · Open Config

优点

  • 最可扩展，未来加任何功能都合理
  • 用户熟悉，无需学习

代价

  • 实现复杂度最高（需下拉菜单组件）
  • 垂直空间占用更多

方案 C：文字按钮 + 更多分隔

保持文字按钮，用逻辑分组和分隔线整理

New Open Save │ New Window │ Win Mode │ About

优点

  • 改动最小，维持现有风格

代价

  • 随着按钮增多仍然会拥挤
  • 10+ 按钮后不可持续
